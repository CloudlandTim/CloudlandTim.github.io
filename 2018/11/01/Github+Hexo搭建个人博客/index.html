Github+Hexo搭建个人博客

Veröffentlicht am 2018-11-01 | Edited on 2018-11-02

个人搭建博客过程完全参考了Dantefung 的教程，仅做了部分主题修改。

环境配置：win10 + Node.js + git + Github + VSCode

完整搭建过程

环境配置

下载安装Git、Node.js和VSCode；目前windows下安装软件真的非常友好了，不需要详细介绍。唯一需要说明的是安装时记得把环境路径给勾选上。

博客搭建

安装Hexo

  1. ‘win+r’ 输入’cmd’ 进入windows 控制台
  2. ‘cd C:\Program Files\nodejs’
    进入Node.js的安装目录
  3. ‘npm install -g hexo’
    安装Hexo
  4. 在任意位置创建你的博客配置文件夹
  5. ‘cd /Blogpath‘
    控制台进入你的博客文件夹，其中/Blogpath代表你的博客配置文件夹地址
  6. ‘hexo init’
    初始化工作目录
  7. ‘npm install’
    安装依赖包,至此一个静态本地页面就搭建完成了
  8. ‘hexo g’
    使用hexo generate生成静态页面
  9. ‘hexo s’
    访问http://localhost:4000 即可在本地测试你的博客

注：如果安装Git时有将Git启动选项加入鼠标右键的话，第1，2步还可以简化为：进入nodejs的安装目录，鼠标右击选择“Git Bash here”

部署博客到Github Pages

  1. 注册Github
  2. 新建一个repository，注意repository名字与Github账户名称一致，然后后缀加上github.io


    注：我在写这篇总结的时候已经创建完毕了，所以仓库里面有内容，正常创建时是一个空项目
  3. 进入该仓库，进行设置，包括站点更新，主题选择


至此，你的github pages就发布成功了。试试在浏览器的地址栏通过博客地址访问一下吧

  1. 回到你的博客配置文件夹，使用编辑器打开配置文件_config.yml

  2. 在文件底部添加Github Pages相关信息

  3. 还是在博客配置文件夹下启动控制台，推送博客内容到GitHub
    ‘npm install hexo-deployer-git –save’ #部署新版本
    ‘hexo g’ #重新生成blog
    ‘hexo d’ #将本地blog部署到github仓库

注：博客内容更新、配置变更都可以通过 ‘hexo g’ + ‘hexo d’ 来完成

至此，刷新一下你的浏览器看看吧。

Hexo主题配置

Hexo 主题网站
挑选一个你喜欢的主题，按照其主页介绍进行安装设置,推荐Hacker，非常简洁，拓展性也好。


主题文件都托管在Github上，包含了详细的安装、配置说明，我们只要下载下来，解压缩到博客配置文件夹/themes文件夹即可

这样就把主题安装好了，然后将博客配置文件_config.yml中theme修改成你下载的主题
然后重来：
‘hexo g’ #重新生成blog
‘hexo d’ #将本地blog部署到github仓库
更新一下后，你就能够看到你所设定的博客风格了。

主题设置

按照所选主题页面介绍进行详细修订，一般是主题文件夹下的配置文件修改相应内容即可。

发布博文

所有的博客均保留在博客文件夹/source/_post中，我们只需要使用编辑器进行书写后，重复’hexo g’ + ‘hexo d’就行啦。
